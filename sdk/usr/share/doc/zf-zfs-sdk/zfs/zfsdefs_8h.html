Flags for opening ZFS scans

Enumerator
OPEN_NOMASKS 

only ZFScan

do not load masks

OPEN_MULTIOPEN 

allow multiple open of a scan (use with care)

OPEN_PHYSICAL 

open in physical taken mode, no wrap

OPEN_NOIMAGE 

open scan without color information

OPEN_IGNORCALTYP 

force open scan if there is no or wrong calib

OPEN_XYZ2POS 

enable support for back transformation from xyz to pixel and line. Need more time for loading scan

OPEN_IGNOR_LEVEL 

do not use leveling system, do not create levelmatrix

OPEN_NEW_MASK 

use new masks older SDK maybe can not read this masks

OPEN_IGNOR_COMPENSATOR 

do not use compensator

OPEN_IGNOR_RFCOMP 

do not use compensated intensity values RfNorm will not work

OPEN_NOCALLBACKS 

do not call callbacks for this scan even if there is a global handler installed

OPEN_NOSCANLIST 

do not put scan into scanlist. Just for internal usage

_OPEN_ENCOPT 

not used

OPEN_USEMATRIX_320 

use registration matrix 320 (final with coordinate system) instead of 280 (without coordinate system) fallback to 320 if 280 does not exist

OPEN_SINGLE 

open only single file if scan has more scans

OPEN_NOPRENEXT 

do not load preview and next line, is much faster but filter will not work (only linebyline)

OPEN_CREATE_TEMP_POS 

create a linepos file in temp folder, usefull for faster read is scan is opend several times

OPEN_READ_MASKS 

read masks created by viewer as well

OPEN_RW 

allow write to file

OPEN_UNAHE 

calculate unahe for viewing, takes time for open scan

OPEN_UNTIL_LAST 

open profiler scans until last availabel scan. Can open scans if rest of scanparts are missing

OPEN_LOGGING 

create logging files
